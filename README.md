# ربات تلگرام «چالش‌ساز» 🔥

ربات تلگرامی که با کمک Claude API چالش‌های وایرال اینستاگرام تولید می‌کند.

## فایل‌ها
- `bot.py` — کد اصلی ربات
- `requirements.txt` — وابستگی‌های پایتون
- `render.yaml` — تعریف سرویس برای دیپلوی خودکار روی Render (اختیاری)

## چطور کار می‌کند؟
1. کاربر یا متن آزاد می‌فرستد (مثلاً «چالش برای پیج ورزشی») یا روی دکمه‌ی
   **🎲 چالش تصادفی** می‌زند.
2. ربات درخواست را همراه با تاریخچه‌ی چالش‌های قبلی همان کاربر (که در حافظه
   نگه داشته می‌شود، نه دیتابیس) به Claude API می‌فرستد.
3. خروجی Claude مستقیماً در تلگرام نمایش داده می‌شود، همراه با همان دکمه برای
   ادامه‌ی بازی.

> **توجه:** چون تاریخچه فقط در حافظه‌ی برنامه نگه داشته می‌شود، با هر بار
> ری‌استارت شدن سرویس (مثلاً دیپلوی جدید، خوابیدن سرویس رایگان و...) پاک
> می‌شود. اگر بعداً خواستی تاریخچه دائمی بماند، باید یک دیتابیس (مثل
> PostgreSQL رایگان روی Render) اضافه کنیم.

## راه‌اندازی روی Render (Background Worker)

### ۱. توکن‌ها را تهیه کن
- **TELEGRAM_BOT_TOKEN**: از [@BotFather](https://t.me/BotFather) در تلگرام
  بگیر (دستور `/newbot`).
- **ANTHROPIC_API_KEY**: از [console.anthropic.com](https://console.anthropic.com)
  بگیر.

### ۲. کد را در یک ریپازیتوری Git قرار بده
این سه فایل (`bot.py`, `requirements.txt`, `render.yaml`) را در یک ریپوی
GitHub/GitLab آپلود کن.

### ۳. سرویس جدید در Render بساز
دو روش داری:

**روش الف — با render.yaml (پیشنهادی):**
1. در داشبورد Render روی **New +** → **Blueprint** بزن.
2. ریپوی خودت را وصل کن؛ Render فایل `render.yaml` را خودش پیدا می‌کند.
3. هنگام ساخت سرویس، مقدار `TELEGRAM_BOT_TOKEN` و `ANTHROPIC_API_KEY` را
   وارد کن (چون `sync: false` هستند، Render از تو می‌خواهد دستی واردشان کنی).

**روش ب — دستی:**
1. در داشبورد Render روی **New +** → **Background Worker** بزن.
2. ریپو را وصل کن.
3. تنظیمات:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python bot.py`
4. در بخش **Environment**، این متغیرها را اضافه کن:
   - `TELEGRAM_BOT_TOKEN`
   - `ANTHROPIC_API_KEY`
   - `CLAUDE_MODEL` (اختیاری، پیش‌فرض `claude-sonnet-4-6`)

### ۴. دیپلوی کن
Render خودش بعد از هر push روی شاخه‌ی اصلی، سرویس را دوباره می‌سازد و اجرا
می‌کند. چون از نوع **Worker** است (نه Web Service)، هیچ نیازی به URL یا
Webhook نیست؛ ربات با Polling مستقیم به تلگرام وصل می‌شود.

## تست لوکال (روی کامپیوتر خودت)
```bash
pip install -r requirements.txt
export TELEGRAM_BOT_TOKEN="توکن_ربات_تلگرام"
export ANTHROPIC_API_KEY="کلید_API_کلود"
python bot.py
```

## نکات توسعه‌ی بعدی (اختیاری)
- اگر بعداً خواستی دکمه‌های موضوعی (ورزشی، خنده‌دار، عاشقانه و...) هم اضافه
  کنی، کافیست در `main_keyboard()` دکمه‌های جدید با `callback_data` متفاوت
  اضافه کنی و در `handle_random_button` بر اساس آن callback_data، پرامپت
  مناسب را به Claude بفرستی.
- اگر خواستی تاریخچه‌ی چالش‌ها بین ری‌استارت‌ها هم باقی بماند، می‌توان به‌جای
  دیکشنری در حافظه از یک دیتابیس (SQLite فایل یا PostgreSQL روی Render)
  استفاده کرد.
